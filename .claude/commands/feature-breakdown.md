## 분해 규칙

아래 규칙을 **모두** 준수하여 실행 계획을 작성하세요.

### 1. 범위
- 이슈에 명시된 내용만 다루세요. 추측으로 범위를 확장하지 마세요.
- 각 단계는 **이슈의 목표에 직접 기여**해야 합니다.

### 2. 단계 크기
- 한 단계는 **하나의 책임**만 가져야 합니다 (SRP).
- 한 번에 리뷰하거나 롤백할 수 있을 만큼 작게 쪼개세요.
- 더 이상 나눌 수 없을 때까지 쪼개되, 과도한 세분화는 피하세요.

### 3. 의존 관계
- 선행 단계가 필요한 경우 `(선행: N단계)` 형식으로 표기하세요.
- 병렬 실행이 가능한 단계는 `(병렬 가능)` 표기를 추가하세요.

### 4. 검증 조건
- 각 단계마다 **완료 여부를 확인하는 방법**을 한 줄로 명시하세요.
- 가능하면 명령어 또는 관찰 가능한 결과로 표현하세요.
