아래 내용을 바탕으로 GitHub 이슈를 생성하세요.

## 요청 내용

$ARGUMENTS

---

## 이슈 작성 규칙

### 1. 이슈 타입 분류
요청 내용을 분석해 다음 중 하나로 분류하세요.

| 타입 | 레이블 | 기준 |
|------|--------|------|
| 기능 추가 | `enhancement` | 새로운 기능 또는 개선 |
| 버그 수정 | `bug` | 오류, 예외 동작, 크래시 |
| 리팩토링 | `refactor` | 동작 변경 없는 코드 개선 |
| 문서 | `documentation` | 문서 추가/수정 |
| 작업/설정 | `chore` | 빌드, 설정, 의존성 등 |

### 2. 제목 규칙
- 명확하고 간결하게 (50자 이내)
- 동사로 시작 (추가, 수정, 개선, 제거 등)
- 타입 prefix 포함: `[feat]`, `[fix]`, `[refactor]`, `[docs]`, `[chore]`

### 3. 본문 구성
이슈 타입에 따라 아래 섹션을 선택해 작성하세요.

**기능 추가 / 리팩토링 / 작업:**
- `## 배경` — 왜 이 작업이 필요한가
- `## 목표` — 완료 시 달라지는 점
- `## 작업 범위` — 포함되는 것 / 포함되지 않는 것
- `## 완료 조건` — 체크리스트 형식

**버그:**
- `## 문제 상황` — 무엇이 잘못되고 있는가
- `## 재현 방법` — 단계별 재현 절차
- `## 기대 동작` vs `## 실제 동작`
- `## 완료 조건` — 체크리스트 형식

---

## 실행 절차

1. 위 규칙에 따라 이슈 제목과 본문을 작성하세요.
2. 현재 저장소의 GitHub remote URL을 확인하세요.
3. 다음 명령으로 이슈를 생성하세요.

```
gh issue create \
  --title "<작성한 제목>" \
  --body "<작성한 본문>" \
  --label "<분류된 레이블>"
```

4. 생성된 이슈 URL을 출력하세요.
